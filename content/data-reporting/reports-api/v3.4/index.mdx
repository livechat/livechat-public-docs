# Introduction

Reports API allows you to **access** and **extract** all the [Reports](https://www.livechat.com/kb/reporting-options-available-in-livechat/) data available in LiveChat.

## Versioning

This document describes the **LiveChat Reports API v3.4**. This is the developer preview version recommended for experimenting with the API capabilities. For production-ready applications, use the currently stable version. [Read more about versioning...](/messaging/#api-versioning)

## Authorization

To call the Reports API, you need to use a Bearer access token. [Learn how to get it.](/authorization/authorizing-api-calls/#implicit-grant)

# Methods

| The API endpoint                                               |
| -------------------------------------------------------------- |
| `https://api.livechatinc.com/v3.4/reports/<resource>/<action>` |

Common headers for all methods:

| Header          | Value              | Required | Notes                                               |
| --------------- | ------------------ | -------- | --------------------------------------------------- |
| `Authorization` | `Bearer <token>`   | Yes      | Your access token                                   |
| `X-API-Version` | `3.4`              | No       | You can specify the API version in the URL instead. |

Additional headers specifically for **POST** method:

| Header         | Value              | Required |
| -------------- | ------------------ | -------- |
| `Content-Type` | `application/json` | Yes      |

## Available parameters and filters

| Parameter                                             | Required | Data type | Notes                                                                                                                                                                                                                                |
| ----------------------------------------------------- | -------- | --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `distribution`                                        | No       | `string`  | Allowed values: `hour`, `day`, `day-hours`, `month` or `year`. Defaults to `day`                                                                                                                                                     |
| `timezone`                                            | No       | `string`  | IANA [Time Zone](http://en.wikipedia.org/wiki/List_of_tz_database_time_zones) (e.g. America/Phoenix). Defaults to the requester's timezone. When requester's timezone is not present then timezone `from` is parsed to get timezone. |
| `filters`                                             | No       | `object`  |                                                                                                                                                                                                                                      |
| `filters.from`                                        | No       | `string`  | Date & time format compatible with RFC3339 with optional resolution of microseconds, `YYYY-MM-DDTHH:MM:SS.ssssssZHH:MM`                                                                                                              |
| `filters.to`                                          | No       | `string`  | Date & time format compatible with RFC3339 with optional resolution of microseconds, `YYYY-MM-DDTHH:MM:SS.ssssssZHH:MM`                                                                                                              |
| `filters.properties.<namespace>.<name>.<filter_type>` | No       | `any`     | **\* described below**                                                                                                                                                                                                               |
| `filters.agents.<string_filter_type>`                 | No       | `any`     | **\*\* described below** `exists` set to `false` will return unassigned chats; `true` will return the assigned ones.                                                                                                                 |
| `filters.tags.<string_filter_type>`                   | No       | `any`     | **\*\* described below**                                                                                                                                                                                                             |
| `filters.sales.<integer_filter_type>`                 | No       | `any`     | **\*\*\* described below**                                                                                                                                                                                                           |
| `filters.goals.<integer_filter_type>`                 | No       | `any`     | **\*\*\* described below**                                                                                                                                                                                                           |
| `filters.surveys.<survey>`                            | No       | `array`   | **\*\*\*\* described below**                                                                                                                                                                                                         |
| `filters.events.types`                                | No       | `array`   | Array of [Event](#events) types. Duplicates are ignored.                                                                                                                                                                             |
| `filters.groups.values`                               | No       | `array`   | Array of group IDs.                                                                                                                                                                                                                  |

**\*)**
`<filter_type>` can take the following values:

- `exists` (`bool`)
- `values` (`type[]` - an array with aspecific type for property: `string`, `int` or `bool`)
- `exclude_values` (`type[]` - array with specific type for property: `string`, `int` or `bool`)

There's only one value allowed for a single property.

**\*\*)**
`<string_filter_type>` can take the following values:

- `exists` (`bool`)
- `values` (`string[]` - an array of strings)
- `exclude_values` (`string[]` - an array of strings)

There's only one value allowed for a single property.

**\*\*\*)**
`<integer_filter_type>` can take the following values:

- `exists` (`bool`)
- `values` (`int[]` - an array of integers)
- `exclude_values` (`int[]` - an array of integers)

There's only one value allowed for a single property.

**\*\*\*\*)**
`<survey>` contains the following fields:

- `type` (`string`) - allowed values: `pre_chat`, `post_chat`
- `answer_id` (`string`)

<CodeResponse version="v3.4" type="reports" title={'Sample filters'} json="filters" />

The JSON payload above translates into following query string:
<CodeSample path={'REQUEST'}>

```shell
?filters.from=2021-03-01T23:59:59-00:00&filters.to=2021-04-13T23:59:59-00:00&filters.properties[routing][continuous].values=true&filters.agents.values=joe@acme.com&filters.agents.values=alice@acme.com&filters.tags.exclude_values=spam&filters.events.types=file
```

Note that when using GET method query string for defining `properties.<namespace>` and `properties.<namespace>.<name>` one should use [].

</CodeSample>

### Agents Chatting Duration

Shows the average chatting duration of agents within a license.

#### Specifics

|                     |                                                                           |
| ------------------- | ------------------------------------------------------------------------- |
| **Method URL**      | `https://api.livechatinc.com/v3.4/reports/chats/agents_chatting_duration` |
| **HTTP method**     | GET, POST                                                                 |
| **Required scopes** | -                                                                         |

#### Request

See [available parameters and filters](#available-parameters-and-filters).

#### Response

| Field                 | Notes                                                  |
| --------------------- | ------------------------------------------------------ |
| `total`               | The total number of chats in the specified date range. |
| `records`             | Contains `distribution` objects, for example, `day`.   |
| `records.day.count`   | The total number of chats `agents` had that `day`.     |
| `records.day.seconds` | The average chat duration `agents` had that `day`.     |

</Text>
<Code>
<CodeSample path={'REQUEST'}>
<Sample>

```shell
curl "https://api.livechatinc.com/v3.4/reports/chats/agents_chatting_duration" \
  -H 'Authorization: Bearer <your_access_token>'
```

</Sample>
</CodeSample>

<CodeResponse type="reports" version="v3.4" json="agentsChattingDuration"/>

</Code>
</Section>

| Field                    | Notes                                                    |
