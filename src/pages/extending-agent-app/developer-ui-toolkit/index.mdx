# Introduction

Developer UI Toolkit is a TypeScript library with React components and views that should be used in applications built for Text Marketplace. It is available in <a href="https://www.npmjs.com/package/@livechat/developer-ui-react" target="_blank">the npm registry</a>.

The library is designed to help developers build beautiful and responsive applications that follow Text's design rules. All the views were built based on the components from the offical Text Design System. The Developer UI Toolkit extends the <Placeholder id="DESIGN_SYSTEM_URL" />, providing new components that were designed specifically for 3rd party application use cases.

# Components

## LiveChat Details

This module provides a context implementation for managing a LiveChat Details Widget using the LiveChat Agent App SDK in a React application. It includes tools for initializing and accessing the details widget and associated customer profile information, ensuring seamless integration within React components.

### LiveChatDetailsProvider

Type: `React.FC<{ children: React.ReactNode | ((context: LiveChatDetailsContext) => React.ReactNode), onSectionLoad?: () => ISection }>`

A provider component that initializes the LiveChat Details Widget and supplies its context to child components.

| Prop | Required | Type | Description |
| ---- | ---- | ---- | ----------- |
| `children` | Yes | <code>React.ReactNode &#124; ((context: LiveChatDetailsContext) => React.ReactNode)</code> | Child components or a render function to access context values.|
| `onSectionLoad` | No | `() => ISection` | Callback function invoked when a section loads. |

<CodeSample path={'LIVECHAT DETAILS PROVIDER'}>

```js
import { LiveChatDetailsProvider } from './LiveChatDetailsContext';

const App = () => {
  return (
    <LiveChatDetailsProvider>
      <MyComponent />
    </LiveChatDetailsProvider>
  );
};
```
</CodeSample>

### useLiveChatDetails

Type: `() => LiveChatDetailsContext`

A custom hook to access the LiveChatDetailsContext. Ensures that the context is properly provided before usage.

<CodeSample path={'LIVECHAT DETAILS HOOK'}>

```js
import { useLiveChatDetails } from './LiveChatDetailsContext';

const CustomerDetails = () => {
  const { widget, customerProfile } = useLiveChatDetails();

  return (
    <div>
      <h1>Widget Name: {widget.name}</h1>
      {customerProfile && <p>Customer: {customerProfile.name}</p>}
    </div>
  );
};
```
</CodeSample>

### withLiveChatDetails

Type: `<T>(Component: React.ComponentType<T & LiveChatDetailsContext>) => React.ComponentType<T>`

A higher-order component (HOC) that injects the `LiveChatDetailsContext` values (`widget` and `customerProfile`) into the wrapped component.

<CodeSample path={'LIVECHAT DETAILS HOC'}>

```js
import { withLiveChatDetails } from './LiveChatDetailsContext';

const CustomerDetails = ({ widget, customerProfile }: LiveChatDetailsContext) => {
  return (
    <div>
      <h1>Widget Name: {widget.name}</h1>
      {customerProfile && <p>Customer: {customerProfile.name}</p>}
    </div>
  );
};

export default withLiveChatDetails(CustomerDetails);
```
</CodeSample>

### Context

`LiveChatDetailsContext`

Type: `React.Context<LiveChatDetailsContext | null>`

A React context that holds the details of the LiveChat Details Widget and the customer profile.

| Prop | Type | Description |
| ---- | ---- | ----------- |
| `widget` | `IDetailsWidget` | The LiveChat Details Widget instance. |
| `customerProfile` | <code>ICustomerProfile &#124; null</code> | The customer profile data, if available. |

## Best practices

<ul>
<li>Always wrap components needing access to <code>...Context</code> with <code>...Provider</code>.</li>
<li>Use the <code>use...</code> hook for function components to simplify accessing context values.</li>
<li>Utilize the <code>with...</code> HOC for class components or components that prefer HOC patterns.</li>
</ul>

## Error handling
<ul>
<li><strong>Missing Context</strong>: The <code>use...</code> hook and the <code>with...</code> HOC throw an error if the <code>...Context</code> is not wrapping the component tree.</li>
<li><strong>Initialization Errors</strong>: If the widget initialization fails, the <code>...Provider</code> renders an <code>AppError</code> component with an error message.</li>
<li><strong>Loading State</strong>: During widget initialization, an <code>AppLoader</code> component is displayed.</li>
</ul>

# Views

## Loading view

Loading view was created to unify the loading state among applications, therefore it doesn't leave much room for customization. It's a simple view with a loading spinner and a message that says `Loading an external application...`.

| Prop | Required | Type | Description |
| ---- | ---- | ---- | ----------- |
| `label` | No | `string` | A message that will be displayed below the loading spinner. <br/>Default: `Loading application...` <br/>In general, the default copy shouldn't be changed unless you're using this component to load some part of your application.|

<CodeSample path={'LOADING VIEW'}>

```js
import { AppLoader } from '@livechat/developer-ui-react';

const Loader = () => (
  <div>
    <AppLoader label="Loading your application..." />
  </div>
);
```

# Errors

The Details Error View is a standardized error component designed to display errors in applications. This component ensures consistency in error handling and provides users with a clear and professional experience when issues arise.

The error message displayed by the Details Error View is a default copy meant to communicate a failure in loading the details view. It helps users understand that something went wrong without overwhelming them with technical details.

# Success

## Success view

The <strong>Success view</strong> component is used to display a success message to the user after the successful installation of the application. It shows the application logo and relevant details to inform the user that the process was completed successfully.

| Prop | Required | Type | Description |
| ---- | ---- | ---- | ----------- |
| `appId` | yes | `string` | The ID of your application. |
| `icon` | yes | `React.ReactNode` | The icon that will be displayed in the success view. |
| `product` | no |<code>'livechat' &#124; 'helpdesk'</code>| The product that the application is integrated with. <br/>Default: 'livechat' |

React component:
<CodeSample path={'SUCCESS VIEW'}>

```js
import { AppSuccessPage } from '@livechat/developer-ui-react';
import { ReactComponent as AppLogo } from './app-logo.svg';

const SuccessView = () => (
  <AppSuccessPage appId="1234" icon={<AppLogo />} product="livechat" />
);
```
</CodeSample>
