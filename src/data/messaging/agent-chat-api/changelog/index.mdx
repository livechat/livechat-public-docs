## [v3.4] - 2021-12-22

### Chats

- There are new pushes:
  - [**chat_deleted**](/messaging/agent-chat-api/v3.4/rtm-pushes/#chat_deleted)
  - [**thread_deleted**](/messaging/agent-chat-api/v3.4/rtm-pushes/#thread_deleted)
  - [**threads_deleted**](/messaging/agent-chat-api/v3.4/rtm-pushes/#threads_deleted)
- The **List Archives** ([Web](/messaging/agent-chat-api/v3.4/#list-archives) & [RTM](/messaging/agent-chat-api/v3.4/rtm-reference/#list-archives)) method:
  - has new filters: `event_types` (which replaces the `events` filter), `greetings`, and `agent_response`.
  - has new filter modifiers: `filters.agents.require_every_value`, `filters.tags.require_every_value`, `filters.sales.require_every_value`, `filters.goals.require_every_value`, and `filters.event_types.require_every_value`.
- Multiple methods no longer require the `chats--my:ro`, `chats--my:rw`, `chats.conversation--my:rw`, and `chats.conversation--my:rw` scopes.
- The **Start Chat** ([Web](/messaging/agent-chat-api/v3.4/#start-chat) & [RTM](/messaging/agent-chat-api/v3.4/rtm-reference/#start-chat)) and **Resume Chat** ([Web](/messaging/agent-chat-api/v3.4/#resume-chat) & [RTM](/messaging/agent-chat-api/v3.4/rtm-reference/#resume-chat)) methods now require that authors (except the requester) of all initial events are listed in the `users` field. Both the events and users have a fixed `visibility` set to `all`.
- The **Start Chat**, **Resume Chat**, **Send Event**, **Send Typing Indicator**, and **Send Rich Message Postback** methods now require either the `chats--all:rw` or `chats--access:rw` scope instead of the `chats.conversation--all:rw` or `chats.conversation--access:rw`.
- The **Deactivate Chat** ([Web](/messaging/agent-chat-api/v3.4/#deactivate-chat) & [RTM](/messaging/agent-chat-api/v3.4/rtm-reference/#deactivate-chat)) method now requires the requester to be present on the list of chat users. You can override it with a new param, `ignore_requester_presence`.

### Chats users

- Users now have a new field, `visibility`.
- The **Add User to Chat** ([Web](/messaging/agent-chat-api/v3.4/#add-user-to-chat) & [RTM](/messaging/agent-chat-api/v3.4/rtm-reference/#add-user-to-chat)) method:
  - now requires an active thread. The `require_active_thread` flag was removed.
  - now requires the requester to be present on the list of chat users. You can override it with a new param, `ignore_requester_presence`.
- The **Remove User from Chat** ([Web](/messaging/agent-chat-api/v3.4/#remove-user-from-chat) & [RTM](/messaging/agent-chat-api/v3.4/rtm-reference/#remove-user-from-chat)) method now requires the requester to be present on the list of chat users. You can override it with a new param, `ignore_requester_presence`.

### Chat access

- The **Transfer Chat** ([Web](/messaging/agent-chat-api/v3.4/#transfer-chat) & [RTM](/messaging/agent-chat-api/v3.4/rtm-reference/#transfer-chat)) method:
  - now requires an active thread.
  - now requires the requester to be present on the list of chat users. You can override it with a new param, `ignore_requester_presence`.
  - had the `force` parameter renamed to `ignore_agents_availability`.
- The **Grant Chat Access** and **Revoke Chat Access** methods were removed. Their use cases are now covered by **Transfer Chat** ([Web](/messaging/agent-chat-api/v3.4/#transfer-chat) & [RTM](/messaging/agent-chat-api/v3.4/rtm-reference/#transfer-chat)). The corresponding pushes, **chat_access_granted** and **chat_access_revoked**, were also removed and replaced by the [**chat_access_updated**](/messaging/agent-chat-api/v3.4/#chat-access-updated) push.

### Configuration

- Updating license resources in [the Configuration API v3](/management/configuration-api/) now results in pushes informing agents about applied changes. These pushes are as follows:
  - [**agent_created**](/messaging/agent-chat-api/v3.4/rtm-pushes/#agent_created)
  - [**agent_approved**](/messaging/agent-chat-api/v3.4/rtm-pushes/#agent_approved)
  - [**agent_updated**](/messaging/agent-chat-api/v3.4/rtm-pushes/#agent_updated)
  - [**agent_suspended**](/messaging/agent-chat-api/v3.4/rtm-pushes/#agent_suspended)
  - [**agent_unsuspended**](/messaging/agent-chat-api/v3.4/rtm-pushes/#agent_unsuspended)
  - [**agent_deleted**](/messaging/agent-chat-api/v3.4/rtm-pushes/#agent_deleted)
  - [**auto_access_added**](/messaging/agent-chat-api/v3.4/rtm-pushes/#auto_access_added)
  - [**auto_access_updated**](/messaging/agent-chat-api/v3.4/rtm-pushes/#auto_access_updated)
  - [**auto_access_deleted**](/messaging/agent-chat-api/v3.4/rtm-pushes/#auto_access_deleted)
  - [**bot_created**](/messaging/agent-chat-api/v3.4/rtm-pushes/#bot_created)
  - [**bot_updated**](/messaging/agent-chat-api/v3.4/rtm-pushes/#bot_updated)
  - [**bot_deleted**](/messaging/agent-chat-api/v3.4/rtm-pushes/#bot_deleted)
  - [**group_created**](/messaging/agent-chat-api/v3.4/rtm-pushes/#group_created)
  - [**group_deleted**](/messaging/agent-chat-api/v3.4/rtm-pushes/#group_deleted)
  - [**group_updated**](/messaging/agent-chat-api/v3.4/rtm-pushes/#group_updated)

### Threads

- There are new properties: `previous_accessible_thread_id` and `next_accessible_thread_id`, which can be returned by the **List Archives** ([Web](/messaging/agent-chat-api/v3.4/#list-archives) & [RTM](/messaging/agent-chat-api/v3.4/rtm-reference/#list-archives)) method.

### Events

- The **Form** ([Web](/messaging/agent-chat-api/v3.4/data-structures/#form) & [RTM](/messaging/agent-chat-api/v3.4/data-structures/#form)) and **Filled Form** ([Web](/messaging/agent-chat-api/v3.4/data-structures/#filled-form) & [RTM](/messaging/agent-chat-api/v3.4/data-structures/#filled-form)) events have a new field, `form_type`.
- All **Events** ([Web](/messaging/agent-chat-api/v3.4/data-structures/#events) & [RTM](/messaging/agent-chat-api/v3.4/data-structures/#events)) had the `recipients` field renamed to `visibility`.
- The **Send Event** ([Web](/messaging/agent-chat-api/v3.4/#send-event) & [RTM](/messaging/agent-chat-api/v3.4/rtm-reference/#send-event)) method now requires the requester to be present on the list of chat users.

### Status

- The **Login** ([RTM](/messaging/agent-chat-api/v3.4/rtm-reference/#login)) method has a new response parameter, `license.organization_id`, which contains the ID of the organization associated with the license you logged into.
- The [**agent_disconnected**](/messaging/agent-chat-api/v3.4/rtm-pushes/#agent_disconnected) push has a new reason, `connection_evicted`.

### Other

- The **Send Typing Indicator** ([Web](/messaging/agent-chat-api/v3.4/#send-typing-indicator) & [RTM](/messaging/agent-chat-api/v3.4/rtm-reference/#send-typing-indicator)) method and [**incoming_typing_indicator**](/messaging/agent-chat-api/v3.4/rtm-pushes/#incoming_typing_indicator) push had the field `recipients` renamed to `visibility`.

## [v3.3] - 2021-03-30

### RTM

- The **General RTM API Request Format** was extended with an optional parameter, `version`. You can now call individual methods on different (higher) versions than the one of your RTM connection.
- The **General Push Format** was extended with the `version` field. To receive pushes from different API versions, subscribe to them via the new, `pushes` parameter in **Login** ([RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#login)).

### Chats

- The **Deactivate Chat** ([Web](/messaging/agent-chat-api/v3.3/#deactivate-chat) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#deactivate-chat)) method had the `chat_id` parameter renamed to `id`.
- The **Follow Chat** ([Web](/messaging/agent-chat-api/v3.3/#follow-chat) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#follow-chat)) method had the `chat_id` parameter renamed to `id`.
- The **Unfollow Chat** ([Web](/messaging/agent-chat-api/v3.3/#unfollow-chat) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#unfollow-chat)) method had the `chat_id` parameter renamed to `id`.
- The **Activate Chat** method was renamed to **Resume Chat** ([Web](/messaging/agent-chat-api/v3.3/#resume-chat) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#resume-chat)).
- The **Start Chat** ([Web](/messaging/agent-chat-api/v3.3/#start-chat) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#start-chat)) and **Resume Chat** ([Web](/messaging/agent-chat-api/v3.3/#resume-chat) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#resume-chat)) methods have a new optional parameter, `active`.
- The **Start Chat** and **Activate Chat** methods now update the requester's `events_seen_up_to` as if all chat events were already seen by the requester. A subsequent call to **Mark Events As Seen** can be skipped in such cases.

### Chat access

- The **Transfer Chat** ([Web](/messaging/agent-chat-api/v3.3/#transfer-chat) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#transfer-chat)) method had the `chat_id` parameter renamed to `id`.

### Events, users & data structures

- The **File** ([Web](/messaging/agent-chat-api/v3.3/data-structures/#file) & [RTM](/messaging/agent-chat-api/v3.3/data-structures/#file)) event has a new parameter for images, `alternative_text`.
- The **Rich Message** ([Web](/messaging/agent-chat-api/v3.3/data-structures/#rich-message) & [RTM](/messaging/agent-chat-api/v3.3/data-structures/#rich-message)) data structure:
  - has a new parameter for images, `alternative_text`,
  - has a new property for buttons, `target`.
- The **Customer** ([Web](/messaging/agent-chat-api/v3.3/data-structures/#customer) & [RTM](/messaging/agent-chat-api/v3.3/data-structures/#customer)) data structure:
  - has a new property for visits, `ended_at`.
  - has new properties: `followed`, `online`, `group_ids`, `state`, `email_verified`.
- In the [**Filled form fields**](/messaging/agent-chat-api/v3.3/data-structures/#filled-form-fields) and [**Form fields**](/messaging/agent-chat-api/v3.3/data-structures/#form-fields) data structures, the `type` property has a new possible value, `subject`. It's a new type that can appear in ticket forms.
- There's a new event type, [**Form**](/messaging/agent-chat-api/v3.3/data-structures/#form) that allows for sending an empty form to fill out.

### Properties

- The **Update Chat Properties** ([Web](/messaging/agent-chat-api/v3.3/#update-chat-properties) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#update-chat-properties)) method had the `chat_id` parameter renamed to `id`.
- The **Delete Chat Properties** ([Web](/messaging/agent-chat-api/v3.3/#delete-chat-properties) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#delete-chat-properties)) method had the `chat_id` parameter renamed to `id`.

### Customers

- The **Get Customer** ([Web](/messaging/agent-chat-api/v3.3/#get-customer) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#get-customer)) method had the `customer_id` parameter renamed to `id`.
- The **Update Customer** ([Web](/messaging/agent-chat-api/v3.3/#update-customer) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#update-customer)) method had the `customer_id` parameter renamed to `id`.
- The **Ban Customer** ([Web](/messaging/agent-chat-api/v3.3/#ban-customer) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#ban-customer)) method had the `customer_id` parameter renamed to `id`.
- There are new methods:
  - **Follow Customer** ([Web](/messaging/agent-chat-api/v3.3/#follow-customer) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#follow-customer)).
  - **Unfollow Customer** ([Web](/messaging/agent-chat-api/v3.3/#unfollow-customer) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#unfollow-customer)).
- There's a new push, [**incoming_customers**](/messaging/agent-chat-api/v3.3/rtm-pushes/#incoming_customers).
- There's a new push, [**customer_unfollowed**](/messaging/agent-chat-api/v3.3/rtm-pushes/#customer_unfollowed), complementary to the **Unfollow Customer** method.
- The **customer_created** and **customer_visit_started** pushes were replaced by the [**incoming_customer**](/messaging/agent-chat-api/v3.3/rtm-pushes/#incoming_customer) push.
- The **customer_visit_ended** push was replaced by the [**customer_left**](/messaging/agent-chat-api/v3.3/rtm-pushes/#customer_left) and [**customer_transferred**](/messaging/agent-chat-api/v3.3/rtm-pushes/#customer_transferred) pushes.
- The **List Customers** ([Web](/messaging/agent-chat-api/v3.3/#list-customers) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#list-customers)) method:
  - has a new parameter, `sort_by`.
  - has new filters: `include_customers_without_chats` and `chat_group_ids`.
  - has a new field, `limited_customers`, added to the response.
- The **List Threads** ([Web](/messaging/agent-chat-api/v3.3/#list-threads) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#list-threads)) method:
  - has new filters: `from` and `to`.
- The [**customer_updated**](/messaging/agent-chat-api/v3.3/rtm-pushes/#customer_updated) push:
  - now returns only the updated data.
  - has a new format; the fields are no longer nested within the `customer` field.

### Status

- The **Login** ([RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#login)) method has new optional parameters:
  - `customer_monitoring_level` - it controls the volume of customer-related pushes sent to the agent.
  - `pushes`- it controls to which pushes the user is subscribed during the current RTM connection.
- There's a new method, **List Routing Statuses** ([Web](/messaging/agent-chat-api/v3.3/#list-routing-statuses) & [RTM](/messaging/agent-chat-api/v3.3/rtm-reference/#list-routing-statuses)), for retrieving the agent's current routing status.

## [v3.2] - 2020-06-18

### Chats

- The **Get Archives** method was renamed to **List Archives** ([Web](/messaging/agent-chat-api/v3.2/#list-archives) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#list-archives)).
- The **List Archives** method:
  - has a new filter, `agents`. It replaces the `agents_ids` filter.
  - has a new filter, `events.types`.
  - has new pagination with hashed page IDs replacing the old pagination format. Pagination is now uniform across the whole Agent Chat API.
  - has a new parameter, `highlights`. It allows you to highlight the match of `filters.query` with an HTML tag.
- The **Get Chats Summary** method was renamed to **List Chats** ([Web](/messaging/agent-chat-api/v3.2/#list-chats) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#list-chats)).
- The **List Chats** method:
  - has a new filter, `include_chats_without_threads`.
  - had the `order` field renamed to `sort_order`.
  - had the `chat.order` field removed from the response.
- The **Get Chat Threads** method evolved into **Get Chat** ([Web](/messaging/agent-chat-api/v3.2/#get-chat) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#get-chat)).
- The **Get Chat** method:
  - allows to retrieve a particular thread from the specified chat.
  - changed its shape; it no longer contains the `order` field and the `chat` key.
  - returns `thread` instead of `threads_summary`.
- The **Close Thread** method was renamed to **Deactivate Chat** ([Web](/messaging/agent-chat-api/v3.2/#deactivate-chat) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#deactivate-chat)). The corresponding push, **thread_closed**, was renamed to [**chat_deactivated**](/messaging/agent-chat-api/v3.2/rtm-pushes/#chat_deactivated).
- The **incoming_chat_thread** push was renamed to [**incoming_chat**](/messaging/agent-chat-api/v3.2/rtm-pushes/#incoming_chat).
- The **incoming_chat** push has new fields, `requester_id` and `transferred_from`.
- There's a new method, **List Threads** ([Web](/messaging/agent-chat-api/v3.2/#list-threads) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#list-threads)).
- There's a new push, [**chat_unfollowed**](/messaging/agent-chat-api/v3.2/rtm-pushes/#chat_unfollowed), complementary to the **Unfollow Chat** method.
- The **Get Chat Threads Summary** method was removed.

### Chat access

- The **Grant Access** method was renamed to **Grant Chat Access** ([Web](/messaging/agent-chat-api/v3.2/#grant-chat-access) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#grant-chat-access)).
- The **Grant Chat Access** method:
  - no longer accepts the `resource` parameter.
  - had its corresponding push, **access_granted**, renamed to [**chat_access_granted**](/messaging/agent-chat-api/v3.2/rtm-pushes/#chat_access_granted).
- The **Revoke Access** method was renamed to **Revoke Chat Access** ([Web](/messaging/agent-chat-api/v3.2/#revoke-chat-access) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#revoke-chat-access)).
- The **Revoke Access** method:
  - no longer accepts the `resource` parameter.
  - had its corresponding push, **access_revoked**, renamed to [**chat_access_revoked**](/messaging/agent-chat-api/v3.2/rtm-pushes/#chat_access_revoked).
- **Transfer Chat** ([Web](/messaging/agent-chat-api/v3.2/#transfer-chat) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#transfer-chat)) can now be used with inactive chats.
- The [**chat_transferred**](/messaging/agent-chat-api/v3.2/rtm-pushes/#chat_transferred) push:
  - is no longer tied directly to **Transfer Chat**. Now, it can be generated when the transfer is implicit, for example, agents are reassigned due to their inactivity.
  - has new fields, `requester_id`, `thread_id`, `reason`, `queue`, and `transferred_to`.
  - no longer has the `type` and `ids` fields.
- The **Set Chat Access** method was removed. Its use cases are now covered by **Transfer Chat** ([Web](/messaging/agent-chat-api/v3.2/#transfer-chat) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#transfer-chat)). The corresponding push, **chat_access_set**, was also removed.

### Chat users

- The **chat_user_added** push was renamed to [**user_added_to_chat**](/messaging/agent-chat-api/v3.2/rtm-pushes/#user_added_to_chat).
- The **user_added_to_chat** push:
  - has new fields: `reason` and `requester_id`.
  - no longer has the `user_type` field.
- The **chat_user_removed** push was renamed to [**user_removed_from_chat**](/messaging/agent-chat-api/v3.2/rtm-pushes/#user_removed_from_chat).
- The **user_removed_from_chat** push:
  - has new fields: `reason` and `requester_id`.
  - no longer has the `user_type` field.
- The **Add User to Chat** method ([Web](/messaging/agent-chat-api/v3.2/#add-user-to-chat) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#add-user-to-chat)) accepts an optional parameter, `require_active_thread`.

### Events

- The [**Thread**](/messaging/agent-chat-api/v3.2/data-structures/#threads) data structure:
  - has new fields: `previous_thread_id`, `next_thread_id`, `queue`, and `queues_duration`.
  - had the `order` and `timestamp` fields replaced with a new one, `created_at` (date & time in microseconds in UTC).
  - had the `restricted_access` field type changed to `string`. It contains the reason for access restriction.
- The [**Filled form fields**](/messaging/agent-chat-api/v3.2/data-structures/#filled-form-fields) data structure has a new parameter, `id` (`fields.answer.id` for the `group_chooser` field type). It's an identifier for each option Customers can choose.
- The [**Chat summary**](/messaging/agent-chat-api/v3.2/data-structures/#chat-summaries) data structure has a new field, `queue`, nested in `last_thread_summary`.

### Properties

- The [**Property**](/messaging/agent-chat-api/v3.2/data-structures/#properties) data structure now has the same format in requests as in responses and pushes.
- The **Update Chat Thread Properties** method was renamed to **Update Thread Properties** ([Web](/messaging/agent-chat-api/v3.2/#update-thread-properties) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#update-thread-properties)). The corresponding push, **chat_thread_properties_updated**, was renamed to [**thread_properties_updated**](/messaging/agent-chat-api/v3.2/rtm-pushes/#thread_properties_updated).
- The **Delete Chat Thread Properties** method was renamed to **Delete Thread Properties** ([Web](/messaging/agent-chat-api/v3.2/#delete-thread-properties) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#delete-thread-properties)). The corresponding push, **chat_thread_properties_deleted**, was renamed to [**thread_properties_deleted**](/messaging/agent-chat-api/v3.2/rtm-pushes/#thread_properties_deleted).

### Thread tags

- The **Tag Chat Thread** method was renamed to **Tag Thread** ([Web](/messaging/agent-chat-api/v3.2/#tag-thread) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#tag-thread)). The corresponding push, **chat_thread_tagged**, was renamed to [**thread_tagged**](/messaging/agent-chat-api/v3.2/rtm-pushes/#thread_tagged).
- The **Untag Chat Thread** method was renamed to **Untag Thread** ([Web](/messaging/agent-chat-api/v3.2/#untag-thread) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#untag-thread)). The corresponding push, **chat_thread_untagged**, was renamed to [**thread_untagged**](/messaging/agent-chat-api/v3.2/rtm-pushes/#thread_untagged).

### Customers

- There's a new method **Get Customer** ([Web](/messaging/agent-chat-api/v3.2/#get-customer) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#get-customer)) that returns info about a given Customer.
- The **Update Customer** ([Web](/messaging/agent-chat-api/v3.2/#update-customer) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#update-customer)) method, which used to return the Customer data structure, now has no response payload.
- Each method, response, and push that deals with a variation of customer `fields` had this field renamed to `session_fields` and its format changed. It's now an array that respects the item order.
- The **Get Customers** method was renamed to **List Customers** ([Web](/messaging/agent-chat-api/v3.2/#list-customers) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#list-customers)).
- In the **List Customers** method, the `order` field was renamed to `sort_order`.

### Status

- The [**Login**](/messaging/agent-chat-api/v3.2/rtm-reference/#login) method:
  - has a new parameter `away`. It was introduced to prevent chat assignments after unexpected reconnects.
  - had the `order` field in the `chat` object removed from the responses.
- The **Update Agent** method was replaced with **Set Routing Status** ([Web](/messaging/agent-chat-api/v3.2/#set-routing-status) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#set-routing-status)). The corresponding push, **agent_updated**, was replaced with [**routing_status_set**](/messaging/agent-chat-api/v3.2/rtm-pushes/#routing_status_set).

### Other

- There's a new method **List Agents For Transfer** ([Web](/messaging/agent-chat-api/v3.2/#list-agents-for-transfer) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#list-agents-for-transfer)).
- There's a new push, [**queue_positions_updated**](/messaging/agent-chat-api/v3.2/rtm-pushes/#queue_positions_updated), with updated positions and wait times for queued chats.
- In **Multicast** ([Web](/messaging/agent-chat-api/v3.2/#multicast) & [RTM](/messaging/agent-chat-api/v3.2/rtm-reference/#multicast)), the `scopes` field was renamed to `recipients`.

## [v3.1] - 2019-09-17

### Added

- **Web API:** The **Send File** method was replaced with **Upload File** ([Web](/messaging/agent-chat-api/v3.1/#upload-file)). The new method should be used along with **Send Event** ([Web](/messaging/agent-chat-api/v3.1/#send-event) & [RTM](/messaging/agent-chat-api/v3.1/rtm-reference/#send-event)), **Start Chat** ([Web](/messaging/agent-chat-api/v3.1/#start-chat) & [RTM](/messaging/agent-chat-api/v3.1/rtm-reference/#start-chat)), or **Activate Chat** ([Web](/messaging/agent-chat-api/v3.1/#activate-chat) & [RTM](/messaging/agent-chat-api/v3.1/rtm-reference/#activate-chat)).

### Changed

- **Web API:** From now on, sending payloads explicitly in the request body is allowed. You no longer have to wrap it up in the `payload` object.
- **Web API:** When using Bot Agents, you need to include the `X-Author-Id` header in requests instead of sending `author_id` in the request body.
- Method **Update Last Seen Timestamp** was transformed into **Mark Events as Seen** ([Web](/messaging/agent-chat-api/v3.1/#mark-events-as-seen) & [RTM](/messaging/agent-chat-api/v3.1/rtm-reference/#mark-events-as-seen)). Now, it accepts the `seen_up_to` parameter (date & time in microseconds in UTC) instead of `timestamp`. Also, it no longer returns `timestamp` in the response.
- Push `last_seen_timestamp_updated` was transformed into [`events_marked_as_seen`](/messaging/agent-chat-api/v3.1/rtm-pushes/#events_marked_as_seen) with `seen_up_to` parameter instead of `timestamp`.
- Fields `order` and `timestamp` in events were replaced with the new field, `created_at` (date & time in microseconds in UTC).
- Method **Create Customer** ([Web](/messaging/agent-chat-api/v3.1/#create-customer) & [RTM](/messaging/agent-chat-api/v3.1/rtm-reference/#create-customer)) now returns only `customer_id` instead of the whole Customer data structure.
- Method **Send Event** ([Web](/messaging/agent-chat-api/v3.1/#send-event) & [RTM](/messaging/agent-chat-api/v3.1/rtm-reference/#send-event)) no longer accepts the `require_active_thread` parameter. Thread is always required to be active unless `attach_to_last_thread` is enabled.
- Method **Send Event** now returns only `event_id` instead of the whole [Event data structure](/messaging/agent-chat-api/v3.1/data-structures/#events).
- Method **Start Chat** ([Web](/messaging/agent-chat-api/v3.1/#start-chat) & [RTM](/messaging/agent-chat-api/v3.1/rtm-reference/#start-chat)) now returns only `chat_id`, `thread_id` and optionally `event_ids` if requested. It used to return the whole Chat data structure.
- Method **Activate Chat** ([Web](/messaging/agent-chat-api/v3.1/#activate-chat) & [RTM](/messaging/agent-chat-api/v3.1/rtm-reference/#activate-chat)) now returns only `thread_id` and optionally `event_ids` if requested. It used to return the whole Chat data structure.

### Removed

- The **Send File** method was removed.
